# Security Policy

Boxup handles backup credentials and restore operations. Please report suspected
security vulnerabilities privately instead of opening a public issue.

Use GitHub's private vulnerability reporting for this repository when available,
or contact the repository owner through their GitHub profile and request a secure
reporting channel. Do not include real passphrases, private keys, repository
credentials, webhook URLs, recovery bundles, archive contents, or production host
details in the initial message.

Include the affected Boxup version or commit, operating system, Borg version, a
minimal reproduction using disposable data, and the security impact. Redact all
production identifiers and secrets.

The latest release and current `main` branch receive security fixes. Older
releases may require upgrading before a fix can be applied.
